Welcome to My Site!

Purpose

Before I get into all the juicy stuff, I want to clarify the exact purpose of this portfolio. I have aimed this blog to provide a quick and precise overview of the fields of expertise, which I posses within the aspect of data science. I will be demonstrating my skills by category in this site, so curious employers who has stumbled into this wicked realm can swiftly locate the skills they are seeking in a candidate.


Structure of the Portfolio

The posts in this portfolio blog are data science projects which have a specific topic in mind. Thus, you will be able to inspect these projects to elude how I have handled the typical challenges associated within a topic of interest.

When you travel this site, you can locate the categories at the top of the page in the navigation bar. The projects within each category are ordered by date. As a final note I will also be tagging each project with more specific topics that outlines which data-tools that are used in a project and common challenges (e.g. merging several datasets, data type conversion, etc...).


Rationale Behind the Categories

I like to define the categories around the data science process. You might be wondering what I am rambling about. But if you look at a typical data analytics project and then subsequently take a step backwards - and then a few more... You will come to a point where most data analytics processes share the same structure. I like to define this structure as the data science process. Allow me to clarify by outlining each of these categories:

1. Setting the Research Goal

  • This is the phase where a question(s) is raised, and the journey begins by seeking the answer to this question from data.
  • While this step is extremely important, it will not be highlighted as a category in my blog since it will implicitly be in every project on this site.

2. Retrieving Data

  • The next step is the elegant art of obtaining the data, but also how to store the data. Sometimes it is required to find the data externally such as web scraping, APIs, NLP or other forms of publicly shared data. Most commonly the data will be available in a normalized database, NoSQL databases or files (txt, csv, excel, etc.).
  • This phase of the process makes it important to hold an in-depth understanding of database normalization and relationships, as well as SQL.
  • Knowledge on how to extract data from various files.
  • Skills in web scraping, API data retrieval and more exotic skills such as natural language processing (NLP).

3. Data Preparation

  • This is a mighty beast and the confrontation is inevitable when handling data. It is often said that data preparation averagely consumes about 80% of a data scientist time. And it cannot be repeated enough: garbage in, equals garbage out. You can further divide the data preparation category into 3 more subcategories - namely, 1) data cleaning, 2) data transformation and 3) combining data.
  • Data Cleaning: this phase aims at removing errors in the data, so it ultimately becomes a true and consistent representation of its origination. This includes topics such as handling missing values, outliers, errors from data entry, physically impossible values, unintentional spaces, typos and errors against the provided codebook.
  • Combining Data: This step is often essential when handling different data-sources. Data varies in size, type and structure, ranging from databases and excel files to text documents. Due to these differences it is essential to know how to combine all the data. This includes topics such as joining datasets, append data, create views and set operators.
  • Data Transformation: the purpose of this phase is to transform the data, so it takes a suitable form for data modeling. This includes topics such as data transformations (log, square-root, etc.), removing redundant variables from the dataset and turning variables into dummies.

4. Data Exploration

  • This is often regarded as the mesmerizing part of data science. Data exploration evolves around making a deep dive into the data by performing various graphical techniques to gain understanding of the data. With an open mind it is possible to grasp interactions between variables. This includes topics such as simple graphs, combined graphs, link / brush and lastly, nongraphical techniques.

5. Data Modeling

  • Here the aim is to build models with the goal of making better predictions, classifying objects or gaining an understanding of the system that you are modeling. It can be regarded as a more focused data exploration, because you know what you are looking for and what you want the outcome to be. This includes topics such as model / variable selection, model execution and model diagnostic / comparisons.

6. Presentation and Automation

  • This is the phase of presenting our findings to the world. The soft skills will be the most useful (yes, they are extremely important). Therefore, a precise and captivating communication is necessary. Sometimes the value that are extracted from a flow of data, are so significant that building an application that automatically updates reports, will be called for. Often this category is naturally found in all my projects, yet special project with interactive dashboards will be located in this category.



Enjoy!

~ Jacob
